You are building RideCheck as a production-grade Next.js 14 App Router (TypeScript) project that runs in Replit with `npm run dev`, and later deploys to Vercel.

ABSOLUTE RULES
- No missing files. No missing imports. No TODOs in core flows.
- Buyers MUST query orders ONLY by customer_id = auth.uid() (never email).
- profiles role is ALWAYS 'customer' on signup (trigger already does this). Never trust metadata role.
- booking_type is ONLY 'self_arrange' | 'concierge'.
- Use Supabase Storage buckets EXACTLY:
  - reports (PDF only, private)
  - inspection-media (images/videos, private)

BOOKING FLOW REQUIREMENTS
Booking types:
1) self_arrange (buyer already arranged appointment)
  - Create order
  - Create Stripe PaymentIntent (automatic capture)
  - Return client_secret to frontend
  - Buyer pays using Stripe Elements
  - Webhook payment_intent.succeeded updates orders.payment_status='paid' and status='payment_received'

2) concierge (RideCheck contacts seller, then requests payment after seller confirms)
  - Create order with payment_status='not_requested'
  - operations/operations_lead updates order status through operations UI
  - When seller confirmed: operations/lead clicks “Send Payment Link”
  - Create Stripe Checkout Session (or Payment Link) and email it (Resend)
  - Webhook marks paid; then ops schedules inspection

EMAIL + SMS
- On order created: email confirmation (Resend)
- On payment requested: email payment link
- On report uploaded: email report-ready with link
- SMS optional but include Twilio client and a toggle; if missing env vars, do not crash—log dev mode.
