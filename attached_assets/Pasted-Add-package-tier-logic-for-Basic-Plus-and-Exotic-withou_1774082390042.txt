Add package tier logic for Basic, Plus, and Exotic without breaking the existing RideCheck booking flow.

GOAL
Use make/model/value/risk rules to determine whether a vehicle should default to Basic, require Plus, or require Exotic.

IMPORTANT
- Do not break existing routing, booking forms, checkout flow, order creation flow, or payment flow
- Do not break auth, admin, ops, Stripe, or current working pages
- Reuse existing package-selection logic if present
- If current package logic exists, extend it safely rather than replacing it entirely
- Keep the user experience clear, mobile-friendly, and easy to understand
- Do NOT create separate Volt or Torque packages in this build
- For this build, EVs and hybrids should be routed into Plus
- Keep the frontend package experience simple for the buyer

PACKAGE RULES
1. Basic = mainstream vehicles with normal inspection complexity
2. Plus = higher-complexity vehicles such as Mercedes-Benz, BMW, Audi, EVs, hybrids, and similar categories
3. Exotic = specialty, ultra-luxury, supercar, collector-risk, or very high-value vehicles

BASIC
- Default tier when no higher-complexity or exotic rule applies

PLUS SHOULD TRIGGER IF ANY OF THESE ARE TRUE
- make is in configured higher-complexity brand list
- vehicle is EV
- vehicle is hybrid
- vehicle falls into a configured diagnostic-complexity category
- vehicle is identified as higher-complexity based on current package rules already in the app

- Mercedes-Benz
- BMW
- Audi
- Volvo

- Jaguar
- Acura
- Infiniti
- Porsche (default to Plus unless overridden to Exotic)
- Lexus higher-complexity models if current logic supports it

EXOTIC SHOULD TRIGGER IF ANY OF THESE ARE TRUE
- make is in exotic brand list
- make/model/trim is in exotic model list
- estimated market value exceeds configured exotic threshold
- vehicle is manually flagged by RideCheck ops/admin

Use an initial exotic brand list like:

- high-end special trims that clearly belong in Exotic

IMPORTANT MODEL/BRAND LOGIC
- Not every luxury brand vehicle should automatically become Exotic
- Porsche should default to Plus unless a model/trim override or value threshold triggers Exotic
- Mercedes-Benz, BMW, and Audi should normally fall into Plus unless a specific exotic override applies
- Everything not matched by special rules should safely remain Basic

MARKET VALUE LOGIC
- Only use estimated market value if there is already a reliable source in the current app
- If no reliable value source exists yet, scaffold the rule structure but do not break the booking flow trying to fetch value data
- Make the exotic value threshold configurable
- If no market value is available, skip value-based escalation and continue using make/model/risk rules

FALLBACK LOGIC
- If make/model/trim cannot be confidently identified, do not block checkout
- Fall back to the safest existing package logic
- If parsing or extraction fails, let the user continue normally
- If no special rule applies, default to Basic
