We need to replace the current flat RideChecker payout model with a dynamic payout system.

Current flat payouts:
- Standard: $50
- Plus: $65
- Premium: $80
- Exotic: $130

We want to convert this to a dynamic payout structure based on:

1. Base Package Rate
2. Distance to inspection
3. Urgency (buyer add-on)
4. Agency tier (Inspector level)
5. Bonus incentives (optional future feature)

---

### NEW PAYOUT STRUCTURE

Each job payout should be calculated as:

Payout = Base Rate 
       + Distance Bonus 
       + Urgency Bonus 
       + Agency Multiplier

---

### 1. BASE RATES

Standard → $45
Plus → $60
Premium → $75
Exotic → $120

(We slightly reduce base to allow incentive layering.)

---

### 2. DISTANCE BONUS

Calculate distance from inspector's location to inspection location.

Add:
0–10 miles → $0
10–25 miles → +$10
25–40 miles → +$20
40+ miles → +$35

Distance must be auto-calculated using stored coordinates.

---

### 3. URGENCY BONUS (if buyer selects urgency add-on)

Same-day → +$15
8-hour priority → +$25
Weekend priority → +$20

Urgency flag already exists in booking model. Use it.

---

### 4. AGENCY / INSPECTOR TIER MULTIPLIER

Add an "inspector_tier" field to inspector profile:

Tier 1 (New) → 1.0x
Tier 2 (Verified) → 1.1x
Tier 3 (Elite) → 1.2x

Final payout = (Base + Distance + Urgency) * Tier Multiplier
